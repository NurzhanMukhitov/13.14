# Анализ размеров секций для iPhone 14

## Характеристики iPhone 14
- **Ширина экрана**: 390px (CSS pixels)
- **Высота экрана**: 844px (CSS pixels)
- **Viewport**: 390 × 844px

---

## Размеры секций

### 1. Hero Section (`.hero`)
```
Внешние размеры:
├─ Ширина: 390px (100%)
└─ Высота: 844px (100vh)

Внутренние размеры (с учётом padding):
├─ Ширина: 350px (390px - 40px padding)
└─ Высота: 804px (844px - 40px padding)

Padding: 20px (все стороны)
Фон: белый (#ffffff)
```

### 2. Visual Section (`.visual`)
```
Внешние размеры:
├─ Ширина: 390px (100%)
└─ Высота: 422px (50vh) ← МЕНЬШЕ чем Hero!

Внутренние размеры для canvas:
├─ Ширина: 390px
└─ Высота: ~342px (422px - 80px padding от .visual-sketch)

Padding: 40px сверху и снизу (от .visual-sketch)
Фон: чёрный (#000000)
```

### 3. About Section (`.about`)
```
Внешние размеры:
├─ Ширина: 390px (100%)
└─ Высота: 844px (100vh) ← ТАКОЙ ЖЕ как Hero

Внутренние размеры (с учётом padding):
├─ Ширина: 350px (390px - 40px padding)
└─ Высота: 724px (844px - 120px padding)

Padding: 60px сверху/снизу, 20px слева/справа
Фон: чёрный (#000000)
```

---

## Сравнение высот секций

| Секция | Высота (vh) | Высота (px) | Отношение к экрану |
|--------|-------------|-------------|-------------------|
| **Hero** | 100vh | 844px | 100% экрана |
| **Visual** | **50vh** | **422px** | **50% экрана** ← В 2 РАЗА МЕНЬШЕ! |
| **About** | 100vh | 844px | 100% экрана |

---

## Вывод

✅ **Вы правы!** У секций действительно разные размеры:

1. **Hero** и **About** — одинаковые (100vh = 844px каждый)
2. **Visual** — в 2 раза меньше (50vh = 422px)

**Общая высота страницы**: 844px + 422px + 844px = **2110px** (2.5 экрана)

---

## Почему так получилось?

В CSS есть медиа-запрос `@media (max-width: 480px)`, который:
- Для **Visual** меняет `min-height` с `70vh` на `50vh`
- Для **Hero** и **About** оставляет `100vh`

Это сделано для оптимизации мобильного просмотра, но создаёт визуальную несбалансированность.
